Global Configuration

Set app-wide defaults for all datepicker instances using the provideDatepickerConfig() provider. This eliminates the need to pass the same props to every datepicker component.

Configuration Options

The following options can be set globally:

  • weekStart - First day of the week (0 = Sunday, 1 = Monday, etc.)
  • minuteInterval - Minute step for time selection (1, 5, 15, 30, etc.)
  • holidayProvider - Global holiday provider for all datepickers
  • yearRange - Year range for year selection dropdown
  • locale - Default locale for date formatting

Component-Level Override

Component-level inputs always override global defaults. This allows you to have global defaults while still customizing individual datepickers when needed.

Format Token Reference:

  • YYYY - 4-digit year (e.g., 2025)
  • YY - 2-digit year (e.g., 25)
  • MM - 2-digit month (e.g., 01-12)
  • M - Month without leading zero (e.g., 1-12)
  • DD - 2-digit day (e.g., 01-31)
  • D - Day without leading zero (e.g., 1-31)
  • hh - 12-hour format with leading zero (e.g., 01-12)
  • h - 12-hour format without leading zero (e.g., 1-12)
  • HH - 24-hour format with leading zero (e.g., 00-23)
  • H - 24-hour format without leading zero (e.g., 0-23)
  • mm - Minutes with leading zero (e.g., 00-59)
  • m - Minutes without leading zero (e.g., 0-59)
  • ss - Seconds with leading zero (e.g., 00-59)
  • s - Seconds without leading zero (e.g., 0-59)
  • A - AM/PM (uppercase)
  • a - am/pm (lowercase)

Timezone Support

Format dates in specific timezones using IANA timezone names. JavaScript Date objects are always stored in UTC internally; the timezone setting only affects how dates are displayed.

html
Important Notes:
  • JavaScript Date objects are always stored in UTC internally
  • The timezone property only affects display formatting, not storage
  • Use IANA timezone names (e.g., 'America/New_York', 'UTC', 'Europe/London')
  • For advanced timezone operations, consider using date-fns-tz or Luxon alongside this library
  • See Timezone Documentation for detailed information
